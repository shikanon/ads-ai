# 广告 TVC 制作网站设计上下文

## Scene

广告运营和创意同学在白天办公环境中使用 27 寸显示器，长时间阅读素材、标签、指标和审计信息，并与跨团队同事截图协作。因此默认采用浅色产品界面，强调信息密度、可读性和稳定的交互反馈。

## Color Strategy

Restrained。暖灰白工作面、偏冷墨蓝文本、单一蓝紫 accent。Accent 只用于主操作、选中态、焦点和关键召回反馈；成功、警告、危险只表达语义状态。

```css
:root {
  --surface: oklch(0.985 0.006 250);
  --surface-2: oklch(0.955 0.008 250);
  --surface-panel: oklch(0.996 0.004 250);
  --text: oklch(0.22 0.025 255);
  --muted: oklch(0.52 0.025 255);
  --border: oklch(0.88 0.012 250);
  --accent: oklch(0.58 0.18 260);
  --success: oklch(0.57 0.13 150);
  --warning: oklch(0.72 0.15 78);
  --danger: oklch(0.58 0.19 28);
}
```

## Typography

使用系统字体栈或 Inter。产品 UI 使用紧凑层级，不使用展示字体。正文说明控制在 65 到 75 字符宽度内，表格和素材行允许更高信息密度。

## Layout

素材库 PC 端采用三栏结构：筛选 Rail、资产主区、Inspector。1440 宽度优先覆盖，筛选和 Inspector 常驻，资产主区使用高密度资产行而不是大卡片墙。移动端不是当前设计目标。

## Components

- 加载态优先使用骨架，不在内容中央使用孤立 spinner。
- 空状态要展示下一步动作或示例资产。
- 状态 chip 需要区分可检索、待处理、失败、阻断和低置信度。
- Inspector 在列表内完成理解和复用，避免把每个轻量动作都推入详情页或弹窗。
- 批量操作条只在多选后出现，并固定在桌面底部。

## Bans

- 不使用渐变文字。
- 不使用装饰性玻璃拟态。
- 不用粗侧边彩条表达状态。
- 不把所有内容做成同尺寸卡片网格。
- 不用模态作为标签校准、确认 AI 标签和轻量操作的首选方案。
