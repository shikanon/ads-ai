# 广告素材库高级视觉方案

## 目标定位

本方案面向广告素材库核心工作台、素材浏览、素材详情、RAG 检索和洞察回流页面，目标是把产品从“素材管理工具”提升为“广告资产智能指挥台”。视觉不追求炫技，而是让用户在 27 寸桌面屏上快速建立信任：素材资产可见、索引逻辑可解释、效果价值可判断、标签治理可执行。

设计注册为 product UI。设计服务任务效率，保留高信息密度、清晰路径和稳定反馈，同时通过更精致的版式、层次、状态系统和数据表达建立高级感。

## 设计概念

### Asset Intelligence Console

视觉关键词：克制、精密、可信、资产感、可解释。

界面像一个广告资产情报控制台，而不是文件夹或上传后台。素材不只是列表项，而是可被检索、复用、回流和审计的资产单元。每个页面都应该回答四个问题：

- 这个素材现在处于什么状态？
- 它为什么被推荐或召回？
- 它带来了什么投放价值？
- 用户下一步可以如何复用或治理它？

## 使用场景

广告运营和创意同学在白天办公环境中使用 27 寸显示器，长时间阅读素材、标签、指标、RAG 证据和审计链路，并经常截图给团队协作。因此主界面采用浅色高密度工作面，避免暗色炫酷风和过强装饰。高级感来自信息秩序、微妙层次、精准语义色和稳定组件，而不是大面积渐变、玻璃拟态或营销式视觉。

## 视觉原则

- 资产先行：首屏优先展示素材价值、效果指标和待处理任务，上传入口保持可见但不抢占中心。
- 解释优先：向量召回、标签过滤、效果加权、RAG 引用来源需要以简洁证据呈现。
- 桌面密度：筛选、资产流、Inspector 同屏存在，减少页面跳转和弹窗。
- 语义状态：状态色只表达可检索、待处理、失败、阻断、低置信度和高效果，不做装饰。
- 稳定节奏：统一组件词汇，避免同一动作在不同页面出现不同按钮形态。
- 克制高级：不使用渐变文字、装饰玻璃卡片、粗侧边彩条、同构卡片墙和无意义动效。

### 色彩使用规则

- Accent 占比控制在 10% 以内，用于“开始检索”“加入素材篮”“确认 AI 标签”等关键动作。
- 高效果素材使用 success chip 和轻量背景，不使用大面积绿色面板。
- 风险素材使用 danger chip、审计提示和阻断说明，不只依赖红色。
- 低置信度和待处理使用 warning，必须附带下一步动作。
- 普通说明、次级按钮和未选中 tab 使用中性层，不引入额外品牌色。

## 字体与排版

字体使用 system-ui 或 Inter。界面应像专业生产工具，而不是品牌落地页。层级通过字号、字重、间距和信息密度建立。

```css
:root {
  --font-sans: Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;
  --text-xs: 0.75rem;
  --text-sm: 0.875rem;
  --text-md: 1rem;
  --text-lg: 1.125rem;
  --text-xl: 1.375rem;
  --text-2xl: 1.75rem;
}
```

### 排版规则

- 页面主标题使用 28px 左右，避免夸张 hero 标题。
- 面板标题使用 16 到 18px，配合简短动作按钮。
- 数据标签使用 12 到 13px，提高密度但保持可读。
- 长说明文本控制在 65 到 75 字符范围内。
- 资产行使用两层文本：标题解释资产内容，次信息解释比例、渠道、入库时间和来源。

## 布局框架

### 全局 Shell

全局结构采用左侧导航、顶部任务栏和主内容区。导航顺序突出素材工作台、素材库、Brief 解析、生成工坊、洞察回流和历史记录。顶部任务栏承载全局搜索、快速入库、素材篮和处理队列。

### 桌面断点

- 1280px：保持核心三栏，不做移动端折叠，允许局部横向滚动。
- 1440px：筛选 Rail 240px，Inspector 360px，资产主区自适应。
- 1728px 以上：筛选 Rail 264px，Inspector 420px，资产主区展示更多指标列。

### 页面节奏

每个页面由“任务意图区、主要工作区、上下文解释区”组成。任务意图区说明用户此刻能完成什么，主要工作区承载素材流或详情，解释区呈现召回、标签、索引、风险和审计。

## 页面视觉方案

### 素材工作台

工作台首屏不做营销 hero，而是做资产总览。顶部是紧凑的资产状态条：可复用素材、待打标、待索引、高效果成品、风险阻断。每个指标都可点击过滤到素材库，不做静态大数字装饰。

中部使用“今日资产流”布局，左侧为推荐复用和最近入库，右侧为处理队列和高效果经验。底部展示改造前后效果：复用率、检索耗时、重复创作减少、高效果素材复用和风险拦截。对比不使用夸张图表，采用细条、趋势点和短证据文案。

高级感来自资产像“可调度资源”一样被组织。每个资产行展示缩略图、状态 chip、标签摘要、CTR/CVR、复用次数和一句推荐理由。

### 素材库浏览页

素材库是核心生产界面，采用三栏资产流：

- 左侧筛选 Rail：库类型、素材类型、渠道、品牌、状态、风险、效果等级。
- 中间资产主区：搜索、召回解释、资产行、批量操作条。
- 右侧 MaterialInspector：选中素材详情、标签治理、索引状态、相似素材和复用动作。

资产行不做大卡片墙，而是类似专业交易台或素材调度台的高密度列表。缩略图占据固定宽度，信息分组明确：内容摘要、标签、效果、状态、动作。选中态使用完整边框和浅色 accent 背景，不用粗侧边彩条。

“索引与召回解释”默认折叠为一行摘要：向量召回数量、标签命中、效果加权和 RAG 引用数量。展开后展示贡献比例和代表证据，帮助用户理解排序而不打断浏览。

### MaterialInspector

Inspector 是高级感的关键。它不是详情页缩小版，而是当前素材的“操作台”。上方展示素材摘要、状态和主动作。中段展示标签分组：内容、业务、效果、合规、场景、行业、管理。底部展示索引状态、审计提示和相似素材。

标签治理操作保持内联：新增、编辑、删除、确认 AI 标签都在 Inspector 内完成。效果标签默认不可直接删除，只能隐藏或排除排序；合规标签删除需要确认文案和审计上下文；locked 标签只读。

### 素材详情页

详情页用于深度解释，而不是替代列表操作。顶部为资产预览和关键指标，主体分为资产信息、标签与索引、投放效果、复用记录。

“入库流水线”使用横向步骤条：接收、清洗、去重、元数据、打标、向量化、索引完成、风险校验。每一步显示状态、时间、产物和失败处理。失败状态必须有原因和下一步动作，例如“重新打标”“提交人工审核”“重新索引”。

### RAG 检索页

检索页采用“问题、证据、回答”结构。顶部输入区展示 query parse：意图、实体、标签和过滤条件。中部左侧是结果资产流，右侧是 RAG 回答面板。回答必须引用素材、标签或经验模板，不能只给泛化建议。

检索结果的高级感来自证据密度：每条结果显示向量相似、标签命中、效果加权和风险状态。用户应该能看出 AI 为什么推荐它。

### 洞察回流页

洞察页展示从高效果素材沉淀出的经验资产。页面使用“方法论、脚本模板、Prompt、失败教训”四类视图。每条洞察绑定来源素材、效果指标、适用场景和复用次数。

视觉上避免知识库文章墙，采用可扫描的知识资产行和右侧预览。经验模板支持加入素材篮、用于 Brief 解析或生成同款。

## 组件规范

### Asset Row

资产行是核心组件，包含：

- 缩略图或文本摘要。
- 标题和次信息。
- 分组标签，不使用无序标签云。
- 状态 chip：searchable、tagging、indexing、blocked、low confidence。
- 指标：CTR、CVR、相似度、复用次数。
- 证据原因：标签命中、向量相似、效果 boost。
- 快捷动作：加入素材篮、找相似、生成同款、查看洞察。

### Status Chip

状态 chip 使用浅底色、细边框和简短文字。禁止只用颜色表达状态。风险、低置信度、失败状态必须有解释或动作。

### Recall Explain

召回解释组件默认折叠，展开后显示：

- 向量召回数量和相似度区间。
- 标签过滤命中项。
- 效果加权原因。
- RAG 引用素材和经验来源。
- 召回来源占比。

### Batch Bar

批量操作条仅在多选后出现，固定在桌面底部。操作包括加入素材篮、批量加标签、批量移除标签、批量确认 AI 标签、批量索引、导出清单。条形视觉保持克制，不遮挡主内容关键行。

### Empty State

空状态必须展示示例资产或下一步动作，例如“先看一组示例资产，了解素材库会如何组织你的创意”。禁止只有插画和“暂无数据”。

## 动效与微交互

动效服务状态，不做装饰。推荐 150 到 220ms，使用 ease-out-quart 或 ease-out-quint。

- 资产行 hover：背景轻微抬升，显示快捷动作。
- Inspector 切换：内容淡入和轻微位移，不改变页面布局稳定性。
- 召回解释展开：高度展开配合透明度变化，避免跳动。
- 批量操作条出现：从底部轻量浮入，不抢焦点。
- 骨架加载：保留资产行结构，让用户预期内容位置。

## 文案风格

文案专业、直接、可执行。每个提示都要说明下一步能做什么。

- 上传入口：把素材放进资产层
- 检索占位：搜索“3 秒强钩子、竖版、家庭伦理、CTR 高于均值”的素材
- 召回摘要：向量召回 42 条，标签命中 6 项，效果加权 3 项
- 低置信度提示：AI 标签置信度偏低，建议确认后参与排序
- 风险提示：该素材已阻断默认检索，可在审计记录中查看原因
- 回流提示：这条成片可沉淀为 finished 素材，并生成一条可复用经验

## 高级感检查清单

- 首屏能看到资产价值，而不是只有上传入口。
- 三栏结构稳定，筛选、资产流和 Inspector 同屏可见。
- 资产行包含缩略图、标签、状态、指标、证据和动作。
- 召回解释可折叠，展开后能解释排序来源。
- 标签治理规则在 UI 中可见，AI、效果、合规、locked 标签行为不同。
- 入库流水线覆盖接收、清洗、去重、元数据、打标、向量化、索引和风险校验。
- 所有 loading 使用骨架或局部状态，不使用孤立 spinner。
- 空状态包含示例资产或下一步动作。
- 不使用渐变文字、装饰玻璃拟态、粗侧边彩条和同构卡片墙。
- 前端验证至少覆盖素材工作台、素材库浏览、素材详情和标签治理。

## 落地优先级

- P0：素材工作台首屏资产总览、三类 demo 资产、改造前后效果区。
- P0：素材库三栏资产流、召回解释、MaterialInspector 和批量操作条。
- P0：素材详情入库流水线和标签治理规则。
- P1：RAG 检索页 query parse、证据结果、引用回答面板。
- P1：洞察回流页知识资产行和来源证据。
- P2：键盘增强、悬停预览、资产对比视图和高级筛选保存。

## 验收方式

- 运行前端 `npm run lint`、`npm run typecheck`、`npm run test`、`npm run build`。
- 截图检查 1440、1728、1920 宽度下三栏结构是否稳定。
- 检查 blocked、低置信度、索引中、可检索、高效果等状态是否语义清晰。
- 检查素材为空或接口失败时是否仍能通过示例资产理解产品价值。
- 检查用户是否能在不进入详情页的情况下完成标签确认、加入素材篮和理解召回原因。
